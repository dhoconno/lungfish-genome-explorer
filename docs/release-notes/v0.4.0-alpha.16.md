# Lungfish 0.4.0-alpha.16

This alpha makes the Sequence menu operational for CLI-backed sequence
transforms and ORF annotation workflows, and fixes several annotation-track
refresh and navigation gaps found during SARS-CoV-2 reference testing.

Changes since v0.4.0-alpha.15:

- Wired Sequence menu reverse-complement and translate actions through the
  existing FASTQ/FASTA operation dialog and CLI-backed execution path.
- Added a standard Find ORFs operation dialog with frame, codon-table,
  minimum-length, partial-ORF, alternative-start, track-name, and track-ID
  options.
- Added CLI-backed ORF annotation creation with reproducibility provenance,
  stored translations, and refresh of the active sequence viewer after track
  mutation.
- Removed the duplicate Annotate Translations command; Find ORFs now creates
  ORF annotations with stored translation data by default.
- Fixed versioned reference sequence name handling so GUI operations against
  views such as `MN908947.3` resolve to the stored reference bundle sequence
  when needed.
- Fixed Go to Gene and Go to Location routing so commands act on the selected
  sequence viewer instead of a stale or unrelated sequence context.
- Added annotation-track deletion from the drawer track menu, including
  provenance for track deletion and cleanup when the last annotation in a track
  is removed.
- Added Track Name and Track ID columns to the bottom annotation drawer so ORF
  and imported annotation tracks can be sorted and filtered directly.
- Fixed ORF and translation annotation visibility for older view-state files so
  newly created ORF tracks render in the sequence viewport without requiring a
  manual reset.
- Kept restriction-site searching out of the Sequence menu and recorded it as a
  future feature request.
- Updated app, CLI, managed-tool lock, help book, release-flow documentation,
  current manual examples, and test version metadata to v0.4.0-alpha.16.

Known follow-up queue:

- Restriction-site search remains a future feature request rather than a hidden
  or partially wired menu item.
- Workflow Builder remains experimental. Continue treating builder-authored
  runs as alpha coverage until broader operation coverage and manual workflow
  testing are complete.
- Existing historical project data may still need backfill when opened; missing
  provenance remains a blocking defect for new scientific import, transform,
  export, and bundle workflows.
- Packaging, release signing, notarization, Sparkle appcast publication, and
  distribution checks remain release gates for every alpha build.
