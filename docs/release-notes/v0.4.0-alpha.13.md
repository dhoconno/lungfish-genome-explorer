# Lungfish 0.4.0-alpha.13

This alpha focuses on making the Workflow Builder testable as an in-app,
CLI-backed workflow surface while also fixing versioned GenBank/GFF reference
handling found during SARS-CoV-2 reference testing.

Changes since v0.4.0-alpha.12:

- Added explicit FASTQ bundle input nodes, richer FASTQ workflow operation
  nodes, a VSP2 workflow template, graph compilation, and native workflow
  execution for builder-authored FASTQ workflows.
- Added CLI support for running Workflow Builder graphs and regression coverage
  for VSP2-style FASTQ bundle workflows, including provenance-producing local
  workflow run bundles.
- Added workflow library management, node inspectors, canvas editing support,
  deletion, connection validation, and menu/dialog integration points so builder
  operations share the same tool configuration surface used elsewhere in the
  app.
- Marked Workflow Builder as an experimental in-progress feature behind the
  app's advanced experimental-features setting.
- Fixed GenBank/GFF reference alias handling so versioned accessions such as
  `MN908947.3` resolve against unversioned bundle sequence names and render
  their annotations together with sequence data.
- Tightened Foundations manual chapters, glossary links, rebranding, screenshots,
  and workflow-builder documentation to match the current app state.
- Updated app, CLI, managed-tool lock, help book, release-flow documentation,
  current manual examples, and test version metadata to v0.4.0-alpha.13.

Known follow-up queue:

- Workflow Builder remains experimental. It now has a usable end-to-end exemplar,
  but the canvas, operation coverage, and run accuracy still need broader manual
  testing before it should be treated as production workflow authoring.
- GenBank/GFF alias handling is covered for versioned accessions, but additional
  multi-record and non-NCBI reference naming cases should be exercised before
  claiming broad annotation-import compatibility.
- Packaging, release signing, notarization, Sparkle appcast publication, and
  distribution checks remain release gates for every alpha build.
