# Lungfish v0.4.0-alpha.6 Release Notes

Previous release: v0.4.0-alpha.5

## Highlights

- Runs multi-sample TaxTriage selections serially, one sample bundle at a time, while preserving a single aggregate batch result for browsing and history.
- Adds serial TaxTriage batch provenance at the aggregate root, with per-sample command, input, output, timing, exit-status, and failure records.
- Imports GFF3 annotation files with valid custom feature types instead of rejecting biologically meaningful non-whitelisted terms.
- Reports GFF3 imports that produce zero usable features as failed operations instead of showing a misleading success.
- Keeps the Annotations drawer column headers and filters visible when too many annotations match to draw, so users can narrow the result set in place.

## Metagenomics And Workflow Changes

- Added `TaxTriageSerialBatchRunner` to execute each selected TaxTriage sample in a deterministic sequence with isolated per-sample output directories.
- Continues remaining TaxTriage samples after an individual sample failure, records failed samples in the aggregate result, and fails the operation only if every sample fails.
- Builds `taxtriage.sqlite` from serial sample output subdirectories so aggregate TaxTriage batch folders open immediately after completion.
- Surfaces TaxTriage partial sample failures in Operation Center logs and completion details.

## Annotation Import And Viewer Fixes

- Accepts syntactically valid custom GFF3 types while continuing to reject empty or malformed feature types.
- Writes import provenance for successful annotation imports and treats zero imported features as a failed import.
- Adds database-backed annotation column filtering so overloaded annotation tables can be narrowed without first drawing every matching row.

## Release And Maintenance

- Updated app, CLI, managed-tool lock, help book, and test version metadata to v0.4.0-alpha.6.
- Added regression coverage for serial TaxTriage execution, serial TaxTriage database import, custom GFF3 feature types, zero-feature import failures, and annotation drawer filtering.
