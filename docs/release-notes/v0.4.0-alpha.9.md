# Lungfish 0.4.0-alpha.9

This alpha fixes classifier read-count stability after row selection and makes
Unique Reads display rules consistent across supported metagenomics views.

Changes since v0.4.0-alpha.8:

- Fixed EsViritu table updates so selecting rows and loading the miniBAM viewer
  no longer rewrites static markdup-derived Unique Reads values.
- Ensured reported EsViritu detections with reads never display `0` Unique
  Reads, including existing batch rows and cached batch manifests.
- Applied the same positive-read Unique Reads invariant to TaxTriage batch and
  organism rows, TaxTriage persisted caches, and NVD result display.
- Preserved existing static TaxTriage Unique Reads values when miniBAM stats are
  emitted for a selected row.
- Added regression coverage for EsViritu and TaxTriage selection/read-stat
  interactions so the table values stay stable while users inspect rows.
