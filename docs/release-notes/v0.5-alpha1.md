# Lungfish 0.5-alpha1

This alpha is a refactoring and runtime-hardening release. It folds in the
multi-wave Claude/Codex review remediation work, raises the minor version for
the larger internal cleanup, and keeps the app focused on reproducible,
project-owned scientific workflows.

Changes since v0.4.0-alpha.16:

## Wave 1: Review Triage And Provenance Foundations

- Triaged the Claude review into implementation waves and recorded the
  accepted plans under `docs/reviews`.
- Tightened provenance materialization for FASTQ-derived inputs so downstream
  commands operate on the intended derived reads instead of silently falling
  back to root payloads.
- Added early runtime/plugin checks around required setup and optional tool
  packs so GUI readiness reflects the actual tool state.
- Started pruning unused module surfaces and clarified app/workflow ownership
  boundaries for later extraction work.

## Wave 2: Boundaries, IO, And Runtime Status

- Moved CLI import workflows toward workflow-layer services instead of app-only
  implementations.
- Replaced fragile source-string tests with behavior-level tests where seams
  were available.
- Hardened IO fixtures and real-file parser tests, including edge cases around
  escaping, compression, and file format detection.
- Improved plugin pack freshness and database recommendations so setup status
  is less stale and more predictable.

## Wave 3: Streaming Parsers And Modal Discipline

- Reworked GenBank and Kraken2 parsing to use streaming, pull-based behavior
  for large files.
- Added ONT import provenance coverage from workflow execution through GUI
  routing.
- Split FASTQ operation execution into smaller planning, CLI invocation,
  output import, cleanup, and provenance rehydration components.
- Migrated additional modal behavior to shared sheet/presenter seams with
  actor-safety tests.

## Wave 4: Dead Workflow Surfaces And Search Correctness

- Removed deprecated workflow UI surfaces and unsupported command aliases that
  were no longer part of the product path.
- Fixed literal SQLite `LIKE` escaping for project search and metadata lookup
  paths.
- Expanded scientific provenance policy gates so missing provenance is caught
  earlier for import, transform, export, and bundle workflows.
- Reduced AppKit modal inventory for reference annotation flows and documented
  remaining exceptions.

## Wave 5: CLI Exit Codes, Metadata, And Modal Reduction

- Normalized several CLI failure paths to specific, script-friendly exit codes.
- Added canonical metadata provenance for bundle metadata changes and synced
  metadata roots.
- Shared sequence extraction bundle output handling between GUI and CLI paths.
- Removed additional `runModal` fallbacks and covered the remaining dialog
  behavior through focused tests.

## Wave 6: Warning Cleanup And Architecture Slices

- Cleared compiler warnings and warning-prone tests left by earlier waves.
- Stabilized primer trim runner behavior and its test fixtures.
- Completed smaller CLI exit-code and IO cleanup items that were intentionally
  deferred from previous waves.
- Tightened AppKit actor hops around modal and operation presentation code.

## Wave 7: Provenance Completion And Dead-Code Pruning

- Added GUI-imported CLI provenance rehydration so final project bundles point
  at final stored payloads, not temporary staging paths.
- Recorded provenance for FASTQ derivatives, bundle merges, metagenomics
  imports, and application export/import workflows.
- Removed more dead workflow validation and IO surfaces.
- Improved NVD, TaxTriage, and classifier command provenance coverage.

## Wave 8: Continuation Cleanup

- Polished the remaining Wave 7 review gaps and follow-up warnings.
- Kept provenance enforcement aligned across CLI, GUI, and workflow-layer
  services.
- Verified the integrated refactor set against the full XCTest and Swift
  Testing suites before moving into launch-readiness extraction.

## Wave 9: Launch Readiness And Controller Extraction

- Extracted large `AppDelegate` responsibilities into reusable coordinators
  for project opening, sample metadata routing, export filenames, sequence
  export panels, graphics export panels, provenance export commands, and file
  panel construction.
- Added `AnalysisResultDisplayRoute` and `ViewerDisplayRoute` seams so result
  and viewer routing can be tested without driving the whole window controller.
- Replaced more direct file-panel construction in app-level flows with small,
  reusable factories.
- Produced a verified debug build from the integrated Wave 9 branch for manual
  testing.

## Wave 10: Final UI, Runtime, And Panel Cleanup

- Fixed Provenance Inspector copyability and selection behavior by using
  selectable wrapping text, consistent row typography, and ANSI-stripped stderr
  display.
- Updated Orient Reads to select reference sequences from project
  `.lungfishref` bundles, matching the read-mapping reference picker behavior.
- Fixed Variant Calling pack readiness so LoFreq, iVar, and Medaka can run
  when their individual tools are installed even if Clair3 is missing.
- Changed the Variant Calling optional pack to use an installable Clair3 pin
  instead of the unavailable `clair3=1.0.10` package.
- Extracted reusable file-panel helpers from Import Center, primer scheme
  import, mapping/workflow-builder panels, viewer drawer panels, Inspector,
  sidebar, Welcome, Database Browser, and Storage Settings flows.

## Recent Release Edits

- Bumped the app, CLI, managed-tool lock, help book, Xcode project metadata,
  release docs, and version-sensitive tests to `0.5-alpha1`.
- Updated the About window copyright to
  `Copyright © 2026 Dave O'Connor`.
- Added this release note as the Sparkle/GitHub release note source for the new
  alpha.

Known follow-up queue:

- Continue treating historical project data without provenance as data that may
  need backfill when opened; missing provenance remains a blocking defect for
  new scientific workflows.
- Workflow Builder remains experimental and should continue to receive narrow
  behavior tests as more operations graduate into reusable workflow seams.
- Packaging, signing, notarization, Sparkle appcast generation, and artifact
  verification remain release gates for every alpha build.
