# Lungfish 0.4.0-alpha.12

This alpha turns a broad integration and documentation pass into a named
checkpoint. It expands the user manual and Read the Docs build, deepens the
workflow builder and local run bundle story, adds mapping and variant-calling
surfaces, hardens provenance expectations across scientific workflows, and
cleans up repository structure for continued agent-heavy development.

Changes since v0.4.0-alpha.11:

- Added Read the Docs deployment scaffolding for the user manual, wired the full
  navigation tree, added PDF output, styled the generated PDF with Lungfish
  branding, and added a PDF download affordance on the manual landing page.
- Expanded the manual draft with tool reference material, provenance
  bibliography notes, workflow chapters, screenshots, and generated/imagegen
  illustrations. The hosted documentation remains primitive, incomplete, and
  not ready for prime time; the release treats it as an alpha documentation
  preview rather than authoritative end-user training material.
- Added development disclaimers and repaired manual theme, banner, font,
  image-resize, and PDF-template issues found while bringing the documentation
  build onto Read the Docs.
- Advanced the workflow builder from catalog and menu exposure toward real local
  execution: graph runs, export validation, workflow diffs, run provenance,
  workflow signing scaffolds, and local workflow run bundles now have broader
  CLI, GUI, and regression-test coverage.
- Added or repaired workflow-builder GUI entry points, headless workflow aliases,
  operation diagnostics, project lock warnings, legacy summary migration, and
  project-lock/migration CLI support.
- Expanded mapping and variant-calling support with mapping read groups,
  extra-argument plumbing for wrapped tools, bcftools integration, per-sample
  variant smart filters, legacy VCF handling, GATK command builders, dry-run
  scaffolding, CLI execution, GUI output attachments, and execution provenance.
- Added phased surfaces for Clair3 and Freyja and tightened related caller,
  lineage, and surveillance workflow expectations without presenting those
  surfaces as fully production-ready.
- Added data-source workflow coverage for first-class CZ-ID import, CZ-ID GUI
  import provenance, FASTQ sample-sheet batch import, GenBank annotation
  sidecars, NCBI GFF3 fetching, empty-annotation handling, NCBI API-key
  settings, live API retry hardening, rate-limit guards, and offline conda UX.
- Added container export and conda lockfile support, offline conda pack aliases,
  conda install-root fixes, managed-tool expectation updates, and documentation
  for headless conda and lockfile workflows.
- Hardened scientific provenance paths across documentation workflows,
  classifier/import/extraction-adjacent surfaces, wrapped-tool extra args,
  local workflow execution, GATK execution, CZ-ID import, and retained fixtures.
- Added fixture provenance auditing that validates field types, content
  semantics, alignment-fixture completeness, stale temporary paths, stale payload
  roots, and retained fixture documentation while ignoring Python cache
  artifacts.
- Completed repository hygiene work by archiving historical plans and specs,
  introducing canonical `agents/` scaffolding, updating active agent archive
  references, removing stale fixture payload paths, reconciling manual
  illustrations, and retiring obsolete illustration ignore rules.
- Added CI/release-script test coverage, stale marker and stale fixture root
  scans, README/manual heading fixes, version reference updates, and the
  v0.4.0-alpha.12 version bump.

Known follow-up queue:

- The Read the Docs manual is now buildable and visible, but it remains an alpha
  documentation preview with incomplete review coverage, rough screenshots, and
  workflows that still need validation against real user tasks.
- Workflow builder, caller, import, and provenance surfaces have moved forward,
  but many are still staged integration points. Treat new scientific workflow
  features as alpha-quality until they have been exercised on representative
  data and reviewed for reproducibility requirements.
- Packaging, release signing, notarization, Sparkle appcast publication, and
  final distribution hardening are not complete in this alpha and remain part of
  the broader release checklist.
