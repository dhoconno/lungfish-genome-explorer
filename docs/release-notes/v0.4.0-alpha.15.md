# Lungfish 0.4.0-alpha.15

This alpha makes provenance a first-class Inspector surface across scientific
bundles and hardens metagenomics provenance coverage for Kraken2, EsViritu, and
TaxTriage results.

Changes since v0.4.0-alpha.14:

- Added a reusable Inspector Provenance tab for scientific bundle inspectors,
  with hierarchical disclosure groups for run summaries, warnings, lineage,
  files, options, runtime identity, and raw JSON.
- Added Inspector-side provenance export affordances so users can inspect and
  export bundle provenance from the right sidebar instead of relying on inline
  one-off provenance summaries.
- Reworked the right Inspector pane to resize like the left sidebar, preserve
  explicit user widths across launches, and avoid clobbering user drag choices
  during ordinary window resizing.
- Fixed Kraken2/Bracken provenance warnings so produced outputs, reports, and
  file metadata are recorded only when they exist and display without
  placeholder "Unknown text" rows.
- Added EsViritu batch/root provenance rollups and backfill so single-sample
  and batch EsViritu result directories expose complete provenance at the
  selected analysis root.
- Added direct TaxTriage pipeline provenance plus app-side backfill for existing
  TaxTriage result directories, including the app-created `taxtriage.sqlite`
  index as a separate Lungfish indexing step without overwriting the original
  Nextflow command lineage.
- Updated the provenance and reproducibility documentation and prototype/spec
  artifacts to match the new Inspector design and stricter provenance contract.
- Updated app, CLI, managed-tool lock, help book, release-flow documentation,
  current manual examples, and test version metadata to v0.4.0-alpha.15.
- Added a post-release GUI polish update: locked shared projects now show a
  persistent read-only banner on open, the Help menu links to the online
  documentation, and **Report an Issue** pre-fills app, Mac, project, and lock
  context.

Known follow-up queue:

- Workflow Builder remains experimental. Continue treating builder-authored
  runs as alpha coverage until broader operation coverage and manual workflow
  testing are complete.
- Existing historical project data may still need backfill when opened; missing
  provenance remains a blocking defect for new scientific import, transform,
  export, and bundle workflows.
- Packaging, release signing, notarization, Sparkle appcast publication, and
  distribution checks remain release gates for every alpha build.
