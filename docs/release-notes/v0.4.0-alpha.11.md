# Lungfish 0.4.0-alpha.11

This alpha turns the first GitHub issue engagement pass into shipped fixes. It
focuses on repository safety for agent-heavy development, better continuity
when reopening work, more expressive table exploration, less intrusive window
behavior, and easier access to operation diagnostics.

Changes since v0.4.0-alpha.10:

- Implemented GitHub Issues #6 through #13 and added narrative issue comments
  describing the disposition and implementation approach for each accepted
  alpha feedback item.
- Switched the repository to a deny-by-default `.gitignore` policy for local
  scratch output and agent runtime state, while preserving the intentionally
  tracked project sources, fixtures, scripts, resources, docs, release notes,
  and provenance test artifacts.
- Removed tracked `.superpowers` scratch log and pid files, with regression
  coverage to prevent agent scratch state from being committed again.
- Expanded saved bundle view state to restore annotation filter text, variant
  filter text, sample display state, and navigation/zoom changes across app
  sessions, with backward-compatible decoding for older saved state.
- Added inverted table filters and AND/OR filter composition through a shared
  `ColumnFilterSet`, then wired those controls through batch classifier tables,
  taxonomy views, viral detection tables, and NAO-MGS result tables.
- Added unit and description tooltips to table headers so dense scientific
  tables can explain column meaning without adding visible clutter.
- Added trackpad pinch zoom to the main sequence viewer and mini BAM viewer,
  including clamped semantic zoom and anchor-stable main-view magnification.
- Added View menu commands to focus the viewer by collapsing both side panes
  and to restore the sidebar and inspector together.
- Changed the operations panel to normal window layering so it no longer floats
  above unrelated windows.
- Added visible operation log actions, including View Log and Reveal in Finder
  from expanded operation rows and context menus.
- Updated app, CLI, managed-tool lock, help book, release documentation, and
  test version metadata to v0.4.0-alpha.11.

Known follow-up queue:

- The issue fixes are intentionally scoped to the accepted alpha feedback in
  #6 through #13. Broader release-signing, Sparkle appcast, and main-branch
  reconciliation work remains governed by the release checklist and not by the
  individual issue threads.
