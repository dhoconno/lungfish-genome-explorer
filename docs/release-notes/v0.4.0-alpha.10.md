# Lungfish 0.4.0-alpha.10

This alpha adds the first end-to-end reads-to-variants path for SARS-CoV-2
teaching workflows, strengthens provenance for newly generated scientific
bundles, and records the first GitHub issue engagement pass for the alpha cycle.

Changes since v0.4.0-alpha.9:

- Added a new "From Reads to Variants" user-manual chapter, including a
  SARS-CoV-2 fixture set, iVar and LoFreq comparison artifacts, glossary terms,
  and a screenshot shotlist for future visual documentation.
- Added an iVar TSV-to-VCF converter with viralrecon parity coverage, codon-aware
  SNP merging, two-sided Fisher exact strand-bias filtering, insertion/deletion
  normalization, and regression fixtures.
- Wired the converter into variant calling through both the GUI and
  `lungfish variants call`, including user-visible options for GFF-aware codon
  merging and strand-bias filtering.
- Added `lungfish bam adopt-mapping` so a mapped BAM can be adopted into a
  Lungfish bundle while preserving the mapping relationship for downstream
  variant workflows.
- Added SRA download fallback support: ENA remains the preferred route, with
  SRA Toolkit `prefetch`/`fasterq-dump` fallback when ENA retrieval fails.
- Added reproducibility provenance for SRA FASTQ downloads, BAM adoption, and
  bundle variant-calling workflows, including final bundle sidecars, resolved
  tool commands, managed conda/runtime identity, output paths, checksums, file
  sizes, exit status, and useful stderr.
- Updated generated iVar VCF headers so `##source=` includes both the iVar
  version and the Lungfish converter version for easier inspection outside a
  `.lungfish*` bundle.
- Added an annotation-database GFF exporter used by the iVar codon-merging path.
- Added end-to-end and parity tests covering reads-to-variants workflows,
  SRA fallback behavior, BAM adoption, provenance attachment, and fixture
  regeneration.
- Added the user-engagement triage process and project lead/orchestrator role
  documents for GitHub Issues.
- Triaged GitHub Issues #6 through #13, labeled them by area/risk, and left
  implementation-focused responses so the alpha backlog has clear next slices.
- Updated app, CLI, managed-tool lock, help book, release documentation, and
  test version metadata to v0.4.0-alpha.10.

Known follow-up queue:

- Issues #6 through #13 are accepted alpha feedback and remain open as
  implementation backlog items rather than being closed as shipped fixes in
  this release.
