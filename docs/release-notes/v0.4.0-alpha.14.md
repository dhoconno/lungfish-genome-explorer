# Lungfish 0.4.0-alpha.14

This alpha focuses on reproducible provenance hardening and same-project
multi-window session support. It also carries forward the experimental Workflow
Builder surface introduced in the previous alpha.

Changes since v0.4.0-alpha.13:

- Added canonical provenance builder, writer, adapter, signing, and envelope
  coverage so scientific outputs consistently record checksums, sizes, tool
  identity, runtime context, command arguments, output manifests, and execution
  status.
- Rehydrated CLI provenance through GUI import/export paths so final Lungfish
  bundles point at their stored payloads instead of temporary staging files.
- Added provenance export reports and stronger regression coverage for FASTQ,
  classifier, extraction, projected provenance, and fixture workflows.
- Fixed classifier extraction provenance so directory-backed classifier results
  record the real input files even when the CLI receives a sentinel result path.
- Added same-project multi-window session support, project window restoration,
  and supporting accessibility/session tests for repeated app-window workflows.
- Updated app, CLI, managed-tool lock, help book, release-flow documentation,
  current manual examples, and test version metadata to v0.4.0-alpha.14.

Known follow-up queue:

- Workflow Builder remains experimental. Continue treating builder-authored
  runs as alpha coverage until broader operation coverage and manual workflow
  testing are complete.
- Provenance coverage is now substantially stricter; new scientific import,
  transform, export, and bundle workflows should treat missing provenance as a
  blocking defect.
- Packaging, release signing, notarization, Sparkle appcast publication, and
  distribution checks remain release gates for every alpha build.
